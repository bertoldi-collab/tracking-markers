# Poetry → uv Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace Poetry with uv as the package manager and build system for `tracking-markers`, with no change to runtime behavior, public API, dependency semantics, or PyPI distribution.

**Architecture:** Convert `pyproject.toml` from the Poetry schema to a standard PEP 621 project table using uv's native `uv_build` backend. Make `pyproject.toml` the single source of the version; `__init__.py` reads it at runtime via `importlib.metadata`. Replace `poetry.lock` with `uv.lock`.

**Tech Stack:** uv 0.11.7, `uv_build` backend, PEP 621 / PEP 735 metadata, pytest.

**Spec:** `docs/superpowers/specs/2026-05-27-migrate-poetry-to-uv-design.md`

---

## Context the engineer needs

- The package uses a **flat layout**: `tracking_markers/` lives at the repo root, not under `src/`. `uv_build` defaults to a `src/` layout, so `module-root = ""` is required.
- The module name auto-normalizes from the project name (`tracking-markers` → `tracking_markers`), so `module-name` does not need to be set.
- There are **2 tests** in `tests/test_tracking_markers.py`. `test_version` asserts `__version__ == '0.9.0'` — this validates the version change. `test_track_points` self-generates its synthetic video into the (gitignored) `data/` dir if missing, so no committed fixtures are needed.
- `.venv` and `dist/` are already in `.gitignore`; no `dist/` files are tracked. `uv build` artifacts therefore will not show up in `git status`.
- Work happens on the existing branch `migrate-poetry-to-uv`.

## File structure

- `pyproject.toml` — full rewrite (Poetry schema → PEP 621 + `uv_build`).
- `tracking_markers/__init__.py` — version now derived from package metadata.
- `poetry.lock` — deleted.
- `uv.lock` — new, generated by uv, committed.
- `README.md` — add a short "Development" section.
